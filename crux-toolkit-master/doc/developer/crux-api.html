• amino_acid: A single amino acid, with three properties: type, mass and a list of chemical modifications.
  • chemical_modification: A modification to an amino acid. It has two properties: name (e.g., acetylation, water-loss, etc.) and mass offset.
  • peptide: A string of amino acids, with properties such as length, charge (possibly "unknown"), mass, and mass-to-charge. The function get_proteins returns a list of proteins in which the peptide occurs.
  • protein : A longer string of amino acids.
  • database: An unordered set of peptides. The primary function is read_database, which reads the database from a given input file.
  • peptide_iterator: An object for traversing the peptides in a database. This object may iterate over only the peptides within a specified mass-to-charge range.
  • ion A (fragment) ion of a peptide, with a pointer to the originating peptide. In addition to mass the ion has four properties: ion type (b or y), cleavage location, charge, a list of chemical modifications (i.e. neutral loss).
  • ion-series A series of fragment ions.
  • peak: A single peak in a mass spectrum, with two properties: mass-to-charge and intensity. The peak also has a (possibly empty) list of associated ions.
  • spectrum: A collection of peaks and some other identifying spectrum information.
  • peak_iterator: An object for traversing the peaks in a spectrum.
  • spectrum_collection: A collection of spectra.
  • spectrum_iterator: An object for traversing the spectra in a spectrum_collection.
  • peptide_spectrum_match: A spectrum and an associated peptide.
  • psm_score_function: A generic scoring routine for peptide_spectrum_matches. Takes as input (1) a SCORE_FUNCTION_T object that specifies the type of score function to be applied, and (2) a single peptide_spectrum_match. Returns a numeric score.

Managing the Crux Web Site

The Crux web site is hosted on GitHub, except for the downloadable releases of the software which are stored on the Noble lab web server.

The content for the web site is stored in the doc/user directory of the Crux repository. The build target Deploy documenation to website on our TeamCity server copies the content from the Crux repository to the Crux website repository. GitHub automatically serves the content of the Crux website repository as http://crux.ms. The association of the Crux website repostory with http://crux.ms can be managed in the GitHub Pages section of settings for the Crux website repository. More information about GitHub support for project web sites can be found here.

The build target Deploy successful build to website on our TeamCity server copies the software distribution files from the TeamCity server to the Noble lab web server. When users agree to the licensing terms, and click on one of the download buttons on the Crux Downloads page, it will activate the PHP script in /noble/www/htdocs/crux-downloads/download.php on the Noble lab web server. This script will initiate the download of the distribution file for the chosen architecture onto the user's machine. Each formal release of Crux will have it's own directory under /noble/www/htdocs/crux-downloads. The daily builds will be placed under /noble/www/htdocs/crux-downloads/daily.

The crux.ms domain name is registered by Charles with Network Soluions. Talk to Charles if you need changes or have questions.

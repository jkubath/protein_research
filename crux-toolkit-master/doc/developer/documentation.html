Documentation

Documentation for commands can be generated using the create-docs command. Run create-docs list to see a list of commands for which documentation can be generated, or create-docs <command> to output the HTML documentation for a command to stdout. The command create-docs default-params can also be run to generate a default parameter file.

The generated HTML is based on a set of templates, which are defined in CreateDocs.cpp. Each template accepts commented tags (i.e. <!-- tag -->) which will be replaced with appropriate values:

  • doc-template: The main template.

    • #NAME#: The name of the command, as specified by the application's getName() function.

    • #DESCRIPTION#: The description of the command, as specified by the application's getDescription() function.

    • #USAGE#: The usage for the command.

    • #INPUTS#: This tag will be replaced by instances of the input template. There will be one instance for each of the command's arguments, as specified by the application's getArgs() function.

    • #OUTPUTSINTRODUCTION#: This tag will be replaced with an introduction to the outputs if the application's needsOutputDirectory() function returns true.

    • #OUTPUTS#: This tag will be replaced by instances of the output template. There will one instance for each of the command's outputs, as specified by the application's getOutputs() function.

    • #OPTIONS#: This tag will be replaced by instances of the option category template. There will one instance for each category of options, when the options specified by the application's getOptions() function are categorized.

  • doc-input-template: The input template. Describes an required argument to the command.

    • #NAME#: The name of the argument.

    • #DESCRIPTION#: The description of the argument, as specified in the Params::Initialize() function.

  • doc-output-template: The output template. Describes an output from the command.

    • #NAME#: The name of the output.

    • #DESCRIPTION#: The description of the output, as specified in by application's getOutputs() function.

  • doc-option-category-template: The option category template. Contains a set of options within a category.

    • #NAME#: The name of the category.

    • #OPTIONS#: This tag will be replaced by instances of the option template. There will be one instance for each option within this category, where the categories are defined in the Params::Categorize() function.

  • doc-option-template: The option template. Describes an optional parameter to the command.

    • #NAME#: The name of the option.

    • #DESCRIPTION#: The description of the option, as specified in the Params::Initialize() function.

    • #VALUES#: The accepted values of the option (e.g. T|F for boolean), as specified in the Params::Initialize() function.

    • #DEFAULT#: The default value of the option, as specified in the Params::Initialize() function.

Note that command usage statements within the Crux application and parameter documentation within written parameter files are based on the same values as the documentation generated by create-docs. To specify that certain text should only appear or be hidden in the HTML documentation, use the tags [[html:<text>]] or [[nohtml:<text>]]. For example, you may want the description of an application to differ between its usage statement and HTML documentation, but the source of both is the application's getDescription() function. You can work around this issue using the above tags, by having the function return a string like:

"This command is used to do [[html:<strong>]]stuff[[html:</strong>]].[[nohtml: This text will not appear in HTML documentation generated by create-docs.]]"

Barista benchmark test

This test checks how well the barista works on increasing samples of ms2 data. The data used comes from increasing in size subsets /net/noble/vol1/data/crux-datasets/2013wu-variation/, specifically the ms2-centroid spectra. The test runs barista on the spectra after each subset is run through tide-search with low resolution and exact p-value. The exact run specifications are shown in ./setup.sh. The performance of barista according to time and max memory usage is shown against the number of input spectra. To change the data set used for benchmarking, one need only modify ./setup.sh ./*_job.sh to change the tide search parameters and location of input files.

You can execute the test by running the script ./dispatch.sh in this directory, which will dispatch a number of jobs to the uge noblelab cluster. After these jobs finish, you can run summarize.py (also in this directory) to parse the output (which currently goes to ~/barista_benchmark/). Assuming that you have gnuplot installed, then your plots should show up in the right-hand set of plots below.
